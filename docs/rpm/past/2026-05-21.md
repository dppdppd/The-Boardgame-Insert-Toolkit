# 2026-05-21

## Accomplished

- Released `v4.10.0`, `v4.11.0`, `v4.11.1`, and `v4.12.0` to GitHub.
- Reworked print-output control around `G_PRINT_TYPES`, `G_PRINT_GROUPS`, and `G_PRINT_BOXES`; removed the superseded isolated-box/divider-output controls.
- Fixed print-group cavity behavior so assigned feature groups produce separate positive insert geometry while the containing box keeps the matching cavity.
- Fixed over-height round feature cavities so they remain negative cavities instead of leaving positive remnants.
- Planned, implemented, tested, documented, packaged, tagged, pushed, and published SVG feature shapes as `v4.12.0`.

## Key Discoveries

- SVG shape configuration is clearest as a block value on `FTR_SHAPE`: `[ FTR_SHAPE, [ SVG, [ SVG_FILE, ... ], [ SVG_WIDTH_MM, ... ], [ SVG_CLEARANCE_MM, ... ] ] ]`.
- SVG scaling does not need source bounds or axis metadata. Requiring `SVG_WIDTH_MM` and using OpenSCAD aspect-ratio preservation keeps the user API smaller and closer to piece measurement.
- SVG cavity chamfers should stay unsupported until a real offset-ring chamfer approach exists; validation now warns and skips them.

## What Didn't Work

- The intermediate `FTR_SHAPE_SVG` subgroup made SVG shapes feel like two independent keys rather than one shape value.
- Earlier source-XY / longest-dimension SVG parameters were unnecessary after switching to required width-based scaling.
