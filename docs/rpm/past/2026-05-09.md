## Key Discoveries

- `DIV_FRAME_RADIUS` was not visibly affecting divider frames because the rounded cube helper call did not pass an explicit rounded-axis vector.
- Small tab sizing was inflated by construction overlap; using an epsilon overlap preserves exact requested tab dimensions.
- `DIV_RAIL_SIZE_XYZ` already resolved omitted rail height to the compartment height internally, but the public API needed a visible `MAX` Z default.
- A BGSD-shaped nested object wrapper such as `[ OBJECT_BOX, [ ... ]]` likely bypasses flat element key lookup and can produce degenerate geometry.
- Sliding-lid detents were weak because the old default was `0.25mm` and generated height was clamped to half wall/lid thickness.
- For lid-cavity lock resistance, `45` degrees gives lock-side relief equal to detent height; `90` degrees preserves the old vertical lock face.
