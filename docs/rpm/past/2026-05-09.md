# 2026-05-09

## Accomplished

- Released BIT `4.3.0` and `4.3.1`.
- Implemented and hardened bay-based generated divider layouts with `DIV_LAYOUT_BAYS` and `DIV_LAYOUT_BAY_SIZE`.
- Refined divider rail controls with `DIV_NO_RAILS_B` and `DIV_RAIL_SIZE_XYZ`, including `MAX` rail height semantics.
- Fixed divider frame/tab sizing behavior, including `DIV_FRAME_RADIUS`, standalone frame size handling, and exact tab sizing.
- Updated divider preview/render placement so previews keep dividers in slots and renders lay them out for printing.
- Doubled test render defaults to `1600x1200`.
- Verified the final release state with full CSG: `72` passed, `0` failed, `18` warnings.

## Key Discoveries

- `DIV_FRAME_RADIUS` was not visibly affecting divider frames because the rounded cube helper call did not pass an explicit rounded-axis vector.
- Small tab sizing was inflated by construction overlap; using an epsilon overlap preserves exact requested tab dimensions.
- `DIV_RAIL_SIZE_XYZ` already resolved omitted rail height to the compartment height internally, but the public API needed a visible `MAX` Z default.
- A BGSD-shaped nested object wrapper such as `[ OBJECT_BOX, [ ... ]]` likely bypasses flat element key lookup and can produce degenerate geometry.

## What Didn't Work

- The nested object wrapper issue was identified but not fixed before session end.
