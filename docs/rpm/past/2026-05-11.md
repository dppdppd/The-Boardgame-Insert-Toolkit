# 2026-05-11 — nested feature groups and copies

## Accomplished

- Shipped `v4.6.1` with practical nested `BOX_FEATURE` geometry support and initial group transform containers.
- Renamed the public transform-group API to `FEATURE_GROUP` / `feature_group`, removed the `BOX_GROUP` alias, and shipped `v4.7.0`.
- Added `FEATURE_COPY` / `feature_copy` plus `FEATURE_REFERENCE` / `feature_reference` for repeating named features or groups, then shipped `v4.8.0`.
- Added focused CSG and render coverage for nested trays, feature groups, feature copies, copied groups, nested copies, and copy validation errors.

## Key Discoveries

- `FEATURE_GROUP` is the clearer public API name because it groups feature/component entries, not boxes.
- `FEATURE_COPY` should apply its own `POSITION_XY` and `ROTATION` as the outer transform while the referenced feature/group supplies local geometry and child layout.
- A copied group is easiest to reason about when the group's own original position is not added twice at the copy site.

## What Didn't Work

- The first group-copy render fixture placed the copied group too close to the source, making the result visually ambiguous; it was widened and re-rendered.
- Malformed enabled copies can trigger OpenSCAD transform warnings after validation; the validation fixture disables malformed copies so diagnostics stay focused.
