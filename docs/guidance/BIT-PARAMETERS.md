# BIT Common Parameters Quick Reference

All parameters for the v4 data array format. See `release/lib/boardgame_insert_toolkit_lib.4.scad` for full definitions.

## Element-level (inside OBJECT_BOX)

`NAME`, `BOX_SIZE_XYZ`, `BOX_FEATURE`, `BOX_LID`, `BOX_NO_LID_B`, `BOX_STACKABLE_B`, `CHAMFER_N`, `ENABLED_B`

## Feature-level (inside BOX_FEATURE)

`FTR_COMPARTMENT_SIZE_XYZ`, `FTR_NUM_COMPARTMENTS_XY`, `FTR_SHAPE` (SQUARE/HEX/HEX2/OCT/OCT2/ROUND/FILLET), `FTR_SHAPE_ROTATED_B`, `FTR_SHAPE_VERTICAL_B`, `FTR_PADDING_XY`, `FTR_PADDING_HEIGHT_ADJUST_XY`, `FTR_MARGIN_FBLR`, `FTR_CUTOUT_SIDES_4B`, `FTR_CUTOUT_CORNERS_4B`, `FTR_CUTOUT_HEIGHT_PCT`, `FTR_CUTOUT_DEPTH_PCT`, `FTR_CUTOUT_WIDTH_PCT`, `FTR_CUTOUT_BOTTOM_B`, `FTR_CUTOUT_BOTTOM_PCT`, `FTR_CUTOUT_TYPE` (INTERIOR/EXTERIOR/BOTH), `FTR_DIVIDERS`, `FTR_SHEAR`, `FTR_FILLET_RADIUS`, `FTR_PEDESTAL_BASE_B`, `CHAMFER_N`, `POSITION_XY`, `ROTATION`

`FTR_DIVIDERS` is an optional subgroup inside `BOX_FEATURE`, shaped like `LABEL`. It carries the divider count for one compartment axis and the normal divider-generation parameters: `DIV_NUM_DIVIDERS`, `DIV_AXIS`, `DIV_RAILS_B`, `DIV_THICKNESS`, `DIV_OUTPUT_ONLY_B`, `DIV_TAB_SIZE_XY`, `DIV_TAB_RADIUS`, `DIV_TAB_CYCLE`, `DIV_TAB_CYCLE_START`, `DIV_TAB_TEXT`, `DIV_TAB_TEXT_SIZE`, `DIV_TAB_TEXT_FONT`, `DIV_TAB_TEXT_SPACING`, `DIV_TAB_TEXT_CHAR_THRESHOLD`, `DIV_TAB_TEXT_EMBOSSED_B`, `DIV_FRAME_SIZE_XY`, `DIV_FRAME_TOP`, `DIV_FRAME_BOTTOM`, `DIV_FRAME_COLUMN`, `DIV_FRAME_RADIUS`, and `DIV_FRAME_NUM_COLUMNS`. `DIV_NUM_DIVIDERS` is the number of evenly spaced generated divider panels. `DIV_AXIS` is `X` or `Y` and defaults to `X`. X-axis dividers are placed at evenly spaced X positions and run front/back; Y-axis dividers are placed at evenly spaced Y positions and run left/right. `DIV_RAILS_B` controls whether matching box-side rail grooves are generated and defaults to `true`. The divider slot gap is inferred from `DIV_THICKNESS + G_TOLERANCE`; optional `DIV_SLOT_DEPTH` controls how far each rail pair projects into the compartment from the side wall and defaults to the wall thickness. Rails live inside the normal feature transform, so box-side rails inherit `POSITION_XY`, `ROTATION`, and `FTR_SHEAR`. Rails and generated loose divider panels are clipped by intersecting the slot slice with the compartment shape, so round/hex/oct side profiles produce matching divider edges. In normal mode, `FTR_DIVIDERS` outputs both the box rails and matching loose divider panels, one panel per requested slot in each repeated compartment. Preview places generated panels in their slots; render lays them out separately for printing. Use `G_PRINT_DIVIDERS` to control generated panel output globally: `true` prints all generated divider panels, `false` prints none, and a string or list of strings prints matching divider/component/box names. Set `G_PRINT_DIVIDERS_ONLY_B` to `true` to suppress boxes and lids and output only the selected generated divider panels. `DIV_OUTPUT_ONLY_B` remains as a per-divider compatibility shortcut for suppressing that parent `OBJECT_BOX` and printing its output-only divider subgroup. Fitted panel extents come from the compartment shape and never exceed the compartment height or side profile. `DIV_TAB_*` tabs are first constrained to the fitted divider side span with their top aligned to the compartment top, then the combined divider is clipped to the compartment profile so tabs do not protrude above or beside the fitted divider. Tab corners are clamped to rounded-rectangle corners, not semicircular caps. `DIV_FRAME_*` keys control optional frame openings. The standalone `OBJECT_DIVIDERS` path remains rectangular and independent. Physical validation warns for malformed counts/axis values, non-positive inferred width/depth, edge-adjacent generated rail grooves, and rail pairs that would overlap across the compartment.

## Lid-level (inside BOX_LID)

`LID_TYPE` (`LID_CAP`/`LID_INSET`/`LID_SLIDING`), `LID_SLIDE_SIDE` (`FRONT`/`BACK`/`LEFT`/`RIGHT`, default `FRONT`), `LID_FRAME_WIDTH` (default wall thickness; 0 omits the extra perimeter frame on patterned, non-solid lids), `LID_SOLID_B`, `LID_HEIGHT`, `LID_FIT_UNDER_B`, `LID_INSET_B` (legacy boolean used when `LID_TYPE` is omitted), `LID_PATTERN_RADIUS`, `LID_PATTERN_N1/N2`, `LID_PATTERN_ANGLE`, `LID_PATTERN_ROW_OFFSET/COL_OFFSET`, `LID_PATTERN_THICKNESS`, `LID_CUTOUT_SIDES_4B`, `LID_LABELS_INVERT_B`, `LID_SOLID_LABELS_DEPTH`, `LID_LABELS_BG_THICKNESS`, `LID_LABELS_BORDER_THICKNESS`, `LID_STRIPE_WIDTH/SPACE`, `LID_TABS_4B`

`LID_SLIDING` creates a sliding panel with matching box rails and a stop on the opposite side from `LID_SLIDE_SIDE`. The rail groove has a horizontal bottom and printable 45-degree top, and the top of that 45-degree wall is the top of the groove. The groove extends halfway into the wall so the lid rests on a wall shelf under the wedge. The lid edge has the matching bevel. Sliding lids use `G_TOLERANCE` for fit: the rail sides get `2 * G_TOLERANCE` clearance each, the stop side gets `G_TOLERANCE`, and the lid panel is shortened vertically for `G_TOLERANCE` above and below. The slide opening has a top-open lead-in rather than a covered slot. Patterned lids include a perimeter frame for rigidity, clipped to the appropriate lid surface; solid lids ignore `LID_FRAME_WIDTH`. Sliding lids include a right-triangle locking detent on the top of the opening-side wall with a matching groove in the underside of the lid. The detent stays on the flat rail shelf after the chamfer, spans that flat width minus `G_TOLERANCE` on both sides, uses `G_DETENT_THICKNESS` for protrusion height, orients the vertical face to resist the lid sliding back out, rotates detached sliding lids 180 degrees for printing so the wedge edges and detent groove face upward, uses `G_LID_THICKNESS` for sliding panel and rail height when set, ignores `LID_HEIGHT` and `LID_FIT_UNDER_B`, and can use the normal solid, pattern, and label parameters.

## Label-level (inside BOX_LID, BOX_FEATURE, or box-level)

`LBL_TEXT`, `LBL_SIZE` (number or AUTO), `LBL_PLACEMENT` (FRONT/BACK/LEFT/RIGHT/FRONT_WALL/BACK_WALL/LEFT_WALL/RIGHT_WALL/CENTER/BOTTOM), `LBL_FONT`, `LBL_DEPTH`, `LBL_SPACING`, `LBL_IMAGE`, `ROTATION`, `POSITION_XY`

## Divider-level

`DIV_TAB_TEXT`, `DIV_TAB_SIZE_XY`, `DIV_TAB_RADIUS`, `DIV_TAB_CYCLE`, `DIV_TAB_CYCLE_START`, `DIV_TAB_TEXT_SIZE/FONT/SPACING`, `DIV_TAB_TEXT_EMBOSSED_B`, `DIV_FRAME_SIZE_XY`, `DIV_FRAME_NUM_COLUMNS`, `DIV_FRAME_COLUMN`, `DIV_FRAME_TOP/BOTTOM/RADIUS`, `DIV_THICKNESS`, `DIV_NUM_DIVIDERS` (only meaningful in `FTR_DIVIDERS`), `DIV_AXIS` (only meaningful in `FTR_DIVIDERS`), `DIV_RAILS_B` (only meaningful in `FTR_DIVIDERS`), `DIV_OUTPUT_ONLY_B` (only meaningful in `FTR_DIVIDERS`)

## Globals (as `[G_*, value]` pairs in data array)

`G_PRINT_LID_B`, `G_PRINT_BOX_B`, `G_PRINT_DIVIDERS`, `G_PRINT_DIVIDERS_ONLY_B`, `G_ISOLATED_PRINT_BOX`, `G_VISUALIZATION_B`, `G_VALIDATE_KEYS_B`, `G_VALIDATE_PHYSICAL_B`, `G_WALL_THICKNESS` (default 2.0mm), `G_LID_THICKNESS` (default follows wall thickness), `G_TOLERANCE`, `G_TOLERANCE_DETENT_POS`, `G_DEFAULT_FONT`, `G_PRINT_MMU_LAYER`

File-scope `$g_*` variables provide defaults; data array entries override them via `Make()`. `G_PRINT_DIVIDERS` accepts `true`, `false`, a divider/component/box name string, or a list of names. Name strings can also use qualified forms like `"box/dividers"` or `"box/component/dividers"`. `G_VALIDATE_KEYS_B` enables key/type validation messages for typos, malformed entries, and wrong value types. `G_VALIDATE_PHYSICAL_B` enables physical validation warnings for known fit and printability risks, including cap/inset/sliding lid fit tolerances, sliding-lid detent sizing/placement, divider slot placement/depth, and fragile side, rounded finger, bottom, and corner cutout dimensions.

Physical validation is enabled by default. For compatibility with older designs, setting only `G_VALIDATE_KEYS_B` to `false` also suppresses physical validation. To run physical checks while suppressing key/type messages, set both `[ G_VALIDATE_KEYS_B, false ]` and `[ G_VALIDATE_PHYSICAL_B, true ]`. To keep key/type messages while suppressing physical warnings, set `[ G_VALIDATE_PHYSICAL_B, false ]`.

## Gotchas

- **`FTR_SHAPE_VERTICAL_B = true` does nothing without an explicit `FTR_SHAPE`.** The default `FTR_SHAPE` is `SQUARE`, and the vertical flag is ignored for square cavities. To get a round vertical cavity (e.g. for a cylindrical token stack), set `FTR_SHAPE = ROUND` and `FTR_SHAPE_VERTICAL_B = true` together. Same applies for vertical hex / oct — pair with `FTR_SHAPE = HEX` (or `HEX2`/`OCT`/`OCT2`).
