# BIT Measurement Checklist

Measure in millimeters. Record whether each value is measured, sourced, assumed, or unknown.

## Game Box

- Interior width, depth, and usable height.
- Board stack width, depth, and height.
- Rulebook and player aid stack width, depth, and height.
- Any required lid clearance for vertical storage.
- Whether trays must sit under a folded board.

## Cards

- Unsleeved card width and height.
- Sleeved card width and height, if applicable.
- Stack height for each card group.
- Whether cards need finger cutouts.
- Whether cards should be removable as gameplay wells.

## Tiles And Boards

- Tile width across flats or points. Record which one you measured.
- Tile thickness.
- Stack height for each tile group.
- Whether hex tiles should be stored flat in rectangular wells or in hex-shaped vertical wells.

## Tokens And Chits

- Diameter or width/depth.
- Thickness.
- Stack height by token type.
- Whether mixed tokens can share a compartment.

## Miniatures, Wooden Pieces, And Odd Shapes

- Maximum width, depth, and height for each piece type.
- Count by color or faction.
- Whether pieces can be mixed or need separated wells.
- Whether fragile parts need extra clearance.

## Printer And Fit

- Preferred wall thickness.
- Fit tolerance for lids.
- Minimum printable detail for labels or patterns.
- Whether lids should be cap, inset, sliding, or omitted.
- Desired clearance around components.

## Before Printing

- Confirm `BOX_SIZE_XYZ` values fit inside the game box.
- Confirm `FTR_COMPARTMENT_SIZE_XYZ` values fit the components.
- Confirm total stacked tray height fits the box.
- Test-print one tight-fitting tray before printing a full insert.
